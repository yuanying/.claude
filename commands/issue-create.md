# GitHub Issue 作成手順

このコマンドは、ユーザーの意見と現在の実装を吸い上げて GitHub Issue を作成します。

## 1. 引数の確認

**$ARGUMENTS**: Issue の目的・要望（オプション）

- 引数がある場合: その内容を Issue の目的として使用
- 引数がない場合: ユーザーに対話形式で質問

### 引数がない場合の質問事項

以下の点についてユーザーに確認する:

1. **Issue の種類**: バグ報告、機能追加、改善提案など
2. **目的・背景**: なぜこの Issue を作成したいのか
3. **対象範囲**: どの部分のコードに関連するか

## 2. 現在の実装の調査

ユーザーの要望に関連するコードを探索し、以下を把握する:

- 関連するファイルのパス
- コードの構造・アーキテクチャ
- 現在の動作の概要

```bash
# リポジトリの状況を確認
git status
git log --oneline -5
```

## 3. Issue 本文の構成

以下のフォーマットで Issue 本文を作成する:

```markdown
## 概要

[ユーザーの要望を整理して記述]

## 現在の実装

[関連コードの概要・構造を記述]
[関連ファイルのパスを列挙]

## 期待される動作 / 変更内容

[具体的な変更点や期待される動作を記述]
```

## 4. Issue の作成

```bash
gh issue create --title "<タイトル>" --body "<本文>"
```

**重要事項**:
- タイトルは簡潔かつ具体的に
- 本文は Markdown 形式で記述
- Issue 作成後、URL を表示してユーザーに共有する

## ワークフロー概要

1. `$ARGUMENTS` を確認し、必要に応じてユーザーに質問
2. 関連するコードを探索・分析
3. Issue のタイトルと本文を生成
4. `gh issue create` で Issue を作成
5. 作成された Issue の URL をユーザーに表示
